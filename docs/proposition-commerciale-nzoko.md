## 1. CONTEXTE

Nzoko Transport opère sur les principales routes du Congo-Brazzaville (RN1, RN2, axes Plateaux et Niari). Aujourd'hui, la vente de billets se fait principalement au guichet, avec les problèmes suivants :

- Files d'attente aux terminus
- Risque de faux billets
- Pas de visibilité sur le remplissage des bus en avance
- Gestion manuelle des places (conflits de sièges)
- Impossibilité de connaître les revenus en temps réel
- Pas de présence digitale face à la concurrence

**Notre solution** : une plateforme de réservation en ligne complète, aux couleurs de Nzoko Transport, accessible depuis le téléphone de chaque client.

---

## 2. LA SOLUTION

### 2.1 Côté Passager (Site web)

| Fonctionnalité | Description |
|---|---|
| Recherche de trajets | Le client choisit son départ, arrivée, date et nombre de passagers |
| Choix du terminus | Brazzaville (Château d'eau, Mpila, Mafouta) / Pointe-Noire (Centre-ville, Nkouikou, Ngoyo) |
| Plan du bus interactif | Le passager voit les places disponibles et choisit la sienne |
| Places Premium | Fenêtres et rangée 1 : tarif majoré (+1 000 FCFA), revenus supplémentaires |
| Paiement Mobile Money | MTN MoMo ou Airtel Money — le client paie depuis son téléphone |
| Billet digital avec QR code | Envoyé par email, imprimable, infalsifiable |
| Compte personnel | Le client retrouve tous ses voyages passés et à venir |

### 2.2 Côté Nzoko Transport (Dashboard de gestion)

| Fonctionnalité | Description |
|---|---|
| Tableau de bord | Réservations du jour, revenus, taux de remplissage |
| Validation des paiements | L'agent confirme ou refuse chaque paiement Mobile Money |
| Scanner QR code | L'agent scanne le billet du passager avec son téléphone avant l'embarquement |
| Validation manuelle | Si problème de caméra, l'agent entre la référence à la main |
| Gestion de la flotte | Ajouter, modifier, désactiver des bus |
| Gestion des lignes | Modifier les trajets, horaires, prix à tout moment |
| Gestion des agents | Créer des comptes pour chaque agent à chaque terminus |
| Sécurité anti-fraude | QR code unique par billet, maximum 3 scans |

### 2.3 Système d'emails automatiques

- Confirmation de réservation envoyée au client par email
- Billet complet avec QR code en pièce jointe
- Le client peut l'afficher sur son téléphone ou l'imprimer

---

## 3. SÉCURITÉ ET ANTI-FRAUDE

| Mesure | Détail |
|---|---|
| QR code unique | Chaque billet a un code impossible à deviner |
| Maximum 3 validations | Au-delà de 3 scans, le billet est refusé |
| Comptes protégés | Chaque agent a son propre accès sécurisé |
| Données chiffrées | Les informations clients sont protégées |
| Historique complet | Qui a scanné, quand, combien de fois — tout est tracé |
